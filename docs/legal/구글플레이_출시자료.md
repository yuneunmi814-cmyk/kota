# 구글 플레이 출시 자료 (스토어 등록 정보 · Data safety · 권한)

> Play Console 제출 시 그대로 복사·선택해 쓰는 자료. 앱 설명에는 PII가 없으므로 공개 가능.
> 개인정보처리방침/약관 전문은 호스팅 필요(아래 6번) — 값은 `docs/legal/*.md` 초안 기반.

## 1. 앱 기본 정보
- **앱 이름**: TravelPack (트래블팩)
- **개발자(표시)명**: 프로젝트윤
- **카테고리**: 여행 및 지역 정보 (Travel & Local)
- **콘텐츠 등급**: 전체이용가 (예상, 아래 4번 IARC 설문)
- **태그/키워드**: 여행, 여행코스, 가이드, 관광, 제주, 국내여행, 체크인

## 2. 짧은 설명 (80자 이내)
```
검증된 여행 코스를 따라가며 체크인하는 패키지여행식 관광 가이드 앱
```

## 3. 자세한 설명 (복사용)
```
여행 준비, 3시간에서 3분으로.

TravelPack은 이것저것 알아보기 번거로운 분들을 위해 '검증된 여행 코스'를 추천하고,
코스를 따라가며 쓰는 가이드 모드와 관광지 큐레이션 정보를 제공하는 패키지여행식 관광 가이드 앱입니다.

■ 이런 분께
- 어디부터 알아봐야 할지 막막한 분
- 대표 명소 위주로 알차게 다니고 싶은 분
- 길찾기·동선 짜기에 시간 쓰기 싫은 분

■ 주요 기능
- 검증된 여행 코스 추천 (한국관광공사 공공데이터 + 에디터 큐레이션)
- 가이드 모드: 현재 위치로 관광지 도착을 자동 체크인, 다음 코스 안내
- 관광지 상세: 운영시간·요금·꿀팁·사진·오디오 가이드(다국어)
- 지역·테마별 탐색과 통합 검색, 저장(북마크)·리뷰
- 크리에이터 마켓플레이스: 나만의 여행팩을 만들어 공개·판매

여행지에서 휴대폰만 따라가면 되는, 가장 쉬운 여행.
```

## 4. 콘텐츠 등급 설문 (IARC) — 예상 답
- 폭력/성적 콘텐츠/욕설/약물/도박: **없음**
- 사용자 간 상호작용(리뷰 등): **있음** (이용자 생성 콘텐츠 — 신고·숨김 기능 보유)
- 위치 공유: 앱 기능 목적의 위치 사용(타 이용자에게 공개 아님)
- 결과: **전체이용가** 예상

## 5. 타깃·광고
- 타깃 연령층: 만 18세 이상(또는 13세 이상) — 마케팅 동의는 선택, 만14세 미만 가입 차단
- 광고 포함: **아니오** (현재 광고 없음)

## 6. 개인정보처리방침 URL (필수)
- Play 등록에 **공개 URL 필수**. `docs/legal/개인정보처리방침.md`를 웹에 게시 후 그 주소 입력.
- 간단 호스팅: GitHub Pages / Netlify에 정적 페이지로. 앱 내 "약관" 링크(`EXPO_PUBLIC_LEGAL_BASE_URL`)와 동일 베이스 사용 권장.

## 7. Data safety (데이터 보안) 양식 — 수집 항목 매핑
※ 실제 동작과 일치해야 함. 모든 전송은 HTTPS 암호화, 계정 삭제(탈퇴) 제공.

| 데이터 유형 | 수집 | 목적 | 필수 | 비고 |
|---|---|---|---|---|
| 위치(대략·정확) | 예 | 앱 기능(가이드 모드 체크인) | 아니오(권한 거부 가능) | 포그라운드만, 6개월 후 좌표 파기 |
| 이메일 주소 | 예 | 계정·인증 | 예 | 소셜 로그인 포함 |
| 이름(닉네임) | 예 | 계정·표시 | 예 | |
| 사용자 콘텐츠(리뷰) | 예 | 앱 기능 | 아니오 | 신고/숨김 운영 |
| 앱 활동(검색·북마크·여행) | 예 | 앱 기능·개인화 | 아니오 | |
| 구매 내역 | 예(유료 이용 시) | 주문/이용권 | 아니오 | 결제수단 정보는 PG가 처리 |
| 기기 식별자(푸시 토큰) | 예 | 알림 | 아니오 | 알림 허용 시 |

- 데이터 암호화 전송: **예**
- 사용자가 데이터 삭제 요청 가능: **예** (앱 내 탈퇴 → 30일 후 완전 파기)

## 8. 권한(Permissions) 선언
- `ACCESS_FINE_LOCATION` / `ACCESS_COARSE_LOCATION` — **포그라운드 전용**, 가이드 모드에서 관광지 도착(체크인) 확인 및 주변 추천.
- **백그라운드 위치 미사용** (별도 강화 심사 불필요).
- `INTERNET`, `POST_NOTIFICATIONS`(알림) — 표준.
- 권한 사용 목적 안내 문구(스토어/런타임):
  ```
  가이드 모드에서 현재 위치로 관광지 도착을 확인하고 주변 관광지를 추천하기 위해 위치 권한을 사용합니다. 위치는 앱 사용 중에만 수집되며 다른 이용자에게 공개되지 않습니다.
  ```

## 10. 출시 트랙 절차
1. 내부 테스트(Internal testing) 트랙에 AAB 업로드(`eas build --profile production` → `eas submit`)
2. 앱 콘텐츠(개인정보처리방침·Data safety·콘텐츠 등급·타깃층·광고·권한) 모두 작성
3. 비공개 테스트(Closed) → 공개 테스트(Open) → 프로덕션 단계적 출시
4. 심사 피드백 대응

## 11. 결제(유료화 시) — 보류 중, 미리 결정 필요
- 디지털콘텐츠(유료 코스)를 앱 내 판매 시 **Google Play 결제 정책** 대상 가능성 → 인앱결제 또는 한국 제3자결제 결정 필요(가이드 3장 참조).
- 현재는 **무료 마켓**으로 출시하고, 유료화 시점에 결제 정책을 확정하는 전략 권장.
